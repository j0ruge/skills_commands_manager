# Checklist Template — CodeRabbit Review

Template for the `coderabbit-review.md` file generated by the skill.

---

## File Structure

```markdown
# CodeRabbit Review — PR #{number}

**Repository**: {owner/repo}
**Branch**: {branch_name}
**Total comments**: {n}
**Date**: {YYYY-MM-DD}

---

## Checklist

- [ ] **1. [CRITICAL] src/path/file.ts:42 — Issue title**
  Issue summary in 1-2 sentences.
  > CodeRabbit suggestion: brief description or code snippet

- [ ] **2. [HIGH] src/other/file.ts:15 — Another title**
  Summary. *(Pre-marked as addressed by CodeRabbit)*

...
```

### After resolution, each item looks like this:

```markdown
- [x] **1. [CRITICAL] src/path/file.ts:42 — Issue title**
  Issue summary.
  - **Status**: Fixed — description of the applied fix

- [x] **2. [HIGH] src/other/file.ts:15 — Another title**
  Summary.
  - **Status**: Already fixed — current code already resolves the issue
```

---

## Ordering

Items are ordered by severity (highest first), then by file:

1. CRITICAL
2. HIGH
3. MEDIUM
4. LOW

Within the same severity, group by file path.

---

## Severity Mapping

| CodeRabbit Marker | Severity | Priority |
|-------------------|----------|----------|
| `🔴` or `Critical` | CRITICAL | 1 (highest) |
| `🟠` or `Major` | HIGH | 2 |
| `🟡` or `Medium` | MEDIUM | 3 |
| `🔵` or `Minor` / `Low` | LOW | 4 (lowest) |
| `Refactor suggestion` | MEDIUM | 3 |
| No marker | MEDIUM | 3 (default) |

CodeRabbit uses both emojis and text labels (`_⚠️ Potential issue_ | _🔴 Critical_`). The skill must handle both formats.

## Final Result Table

At the end of the file, maintain:

```markdown
---

## Final Result

| Status | Count |
|--------|-------|
| Fixed | {n} |
| Already fixed | {n} |
| Not applicable | {n} |
| Pending | {n} |
| **Total** | **{n}** |

### Regression Tests

- **Command**: `{test_command}`
- **Result**: {All passed (N tests) | N failures}
```

---

## Special Cases

- **Comments without file/line** (PR-level summaries): include in checklist without `file:line`, use category "GENERAL"
- **Deleted files**: mark `[x] File removed — not applicable`
- **Multi-line suggestions**: include the full suggestion block, indented under the item
- **Duplicate comments**: deduplicate by file + line + first 100 chars of body. Note deduplication in header
- **Comments with "Also applies to: lines X-Y"**: treat as a single item covering multiple locations
