# Checklist Template — PR Review Resolution

Template for the `{reviewer}-review.md` files generated by the skill. Each reviewer gets its own file.

---

## File Structure

```markdown
# {Reviewer Name} Review — PR #{number}

**Repository**: {owner/repo}
**Branch**: {branch_name}
**Reviewer**: {reviewer bot login}
**Total comments**: {n}
**Date**: {YYYY-MM-DD}

---

## Checklist

### CRITICAL

- [ ] **1. [CRITICAL] src/path/file.ts:42 — Issue title**
  Issue summary in 1-2 sentences.
  > Suggestion: brief description or code snippet

### HIGH

- [ ] **2. [HIGH] src/other/file.ts:15 — Another title**
  Summary. *(Pre-marked as addressed)*

### MEDIUM

...

### LOW

...
```

### After resolution, each item looks like:

```markdown
- [x] **1. [CRITICAL] src/path/file.ts:42 — Issue title**
  Issue summary.
  - **Status**: Fixed — description of the applied fix

- [x] **2. [HIGH ↑ from MEDIUM] src/other/file.ts:15 — Another title**
  Summary. Severity recalibrated: breaks feature flow (handleComplete silently fails).
  - **Status**: Fixed — removed status from SKIP_FIELDS

- [x] **3. [MEDIUM] src/services/exportService.ts:173 — Error message in English**
  Summary.
  - **Status**: Already fixed — see coderabbit-review.md #11
```

When severity is recalibrated from the reviewer's original, note it as `[NEW_SEVERITY ↑ from ORIGINAL]` and explain why in the summary.

### Minimal file for reviewers with zero findings:

```markdown
# Gemini Review — PR #3

**Repository**: owner/repo
**Reviewer**: gemini-code-assist[bot]
**Date**: 2026-04-12

No actionable findings — reviewer approved without issues.
```

---

## Ordering

Items are ordered by severity (highest first), then by file:

1. CRITICAL
2. HIGH
3. MEDIUM
4. LOW

Within the same severity, group by file path.

---

## Severity Mapping

### CodeRabbit

| Marker | Severity |
|--------|----------|
| `🔴` or `Critical` | CRITICAL |
| `🟠` or `Major` | HIGH |
| `🟡` or `Medium` | MEDIUM |
| `🔵` or `Minor` / `Low` | LOW |
| `Refactor suggestion` | MEDIUM |
| `Potential issue` | Depends on paired severity emoji |
| No marker | MEDIUM (default) |

### Copilot / Codex / Unknown

No severity markers — default all to MEDIUM.

### Gemini

Uses text labels: `critical`, `high`, `medium`, `low` (case-insensitive).

## Special Cases

- **Comments without file/line** (PR-level summaries): include in checklist without `file:line`, use category "GENERAL"
- **Deleted files**: mark `[x] File removed — not applicable`
- **Multi-line suggestions**: include the full suggestion block, indented under the item
- **Duplicate comments**: deduplicate by file + line + first 100 chars of title. Note deduplication in header
- **Comments with "Also applies to: lines X-Y"**: treat as a single item covering multiple locations
- **Same root cause across items**: keep separate checklist entries (different file:line), but note "Related to item #N" in the summary
